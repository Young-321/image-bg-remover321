### 2.2 功能详细设计

#### 功能 UR-001: 用户类型检测
**需求**: 检测用户类型（未登录/已登录/付费）

**设计方案**:
- 使用React Context + localStorage实现
- 未登录用户: localStorage中无用户信息
- 登录用户: localStorage中有用户信息
- 付费用户: 用户信息中包含付费套餐标识

**技术实现**:
```typescript
type UserType = 'guest' | 'logged_in' | 'paid';

interface User {
  id: string;
  email: string;
  name: string;
  userType: UserType;
  quota: {
    guest: number;      // 未登录配额
    loggedIn: number;   // 登录用户配额
    paid: number;       // 付费配额
  };
}
```

**API设计**:
- `GET /api/user/type` - 获取当前用户类型
- `GET /api/user/info` - 获取用户信息

#### 功能 UR-002 ~ UR-004: 登录注册
**需求**: Google登录、自动注册、登录状态保持、退出登录

**设计方案**:
- 延续现有Google OAuth实现
- 使用AuthContext统一管理
- 登录成功后自动初始化配额

**用户流程**:
1. 用户点击"使用Google登录"
2. 跳转到Google OAuth页面

#### 功能 BR-003: 登录配额管理
**需求**: 管理已登录用户每月5次免费配额

**数据结构**:
```typescript
interface LoggedInQuota {
  monthly: {
    year: number;      // 年份
    month: number;     // 月份 (1-12)
    total: number;     // 总配额(5)
    used: number;      // 已用
    remaining: number; // 剩余
  };
  lastResetAt: number; // 最后重置时间
}
```

**配额重置逻辑**:
```typescript
function checkAndResetQuota(userId: string): LoggedInQuota {
  const now = new Date();
  const currentYear = now.getFullYear();
  const currentMonth = now.getMonth() + 1;

  // 从数据库获取用户配额
  const quota = getQuotaFromDB(userId);

  // 检查是否需要重置
  if (!quota.monthly || 
      quota.monthly.year < currentYear ||
      quota.monthly.month < currentMonth) {
    
    // 重置配额
    quota.monthly = {
      year: currentYear,
      month: currentMonth,
      total: 5,
      used: 0,
      remaining: 5
    };
    quota.lastResetAt = Date.now();
    
    // 保存到数据库
    saveQuotaToDB(userId, quota);
  }

  return quota;
}
```

**月度重置任务**:
- 使用Cron Job定时任务
- 每月1日 00:00:00执行
- 批量重置所有登录用户配额

#### 功能 BR-007: 配额扣除优先级
**需求**: 优先扣除免费配额，用完后扣除付费配额

**扣除逻辑**:
```typescript
async function deductQuota(userId: string): Promise<boolean> {
  const user = await getUser(userId);
  const quota = await getQuota(userId);

  // 1. 优先使用登录用户免费配额
  if (quota.loggedIn.remaining > 0) {
    quota.loggedIn.remaining--;
    quota.loggedIn.used++;
    await recordTransaction(userId, 'usage_logged_in', 1);
    await saveQuota(userId, quota);
    return true;
  }

  // 2. 再使用付费配额
  if (quota.paid.remaining > 0) {
    quota.paid.remaining--;
    quota.paid.used++;
    await recordTransaction(userId, 'usage_paid', 1);
    await saveQuota(userId, quota);
    return true;
  }

  // 3. 配额不足
  return false;
}
```

**配额扣除流程**:
1. 用户上传图片
2. 检查用户类型
3. 未登录用户: 检查guest配额
4. 登录用户: 检查loggedIn配额
5. 付费用户: 先检查loggedIn，再检查paid
6. 配额足够: 扣除，处理图片
7. 配额不足: 显示引导提示

## 7. 数据库设计

### 7.1 用户表 (users)
```sql
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  email TEXT UNIQUE NOT NULL,
  name TEXT,
  avatar_url TEXT,
  provider TEXT NOT NULL,
  provider_id TEXT NOT NULL,
  user_type TEXT NOT NULL DEFAULT 'logged_in',
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  last_login_at INTEGER NOT NULL
);
```

### 7.2 用户配额表 (user_quotas)
```sql
CREATE TABLE user_quotas (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL UNIQUE,
  
  -- 未登录用户配额(不重置)
  guest_total INTEGER DEFAULT 3,
  guest_used INTEGER DEFAULT 0,
  guest_remaining INTEGER DEFAULT 3,
  
  -- 登录用户月度配额(每月重置)
  logged_in_monthly_year INTEGER,
  logged_in_monthly_month INTEGER,
  logged_in_monthly_total INTEGER DEFAULT 5,
  logged_in_monthly_used INTEGER DEFAULT 0,
  logged_in_monthly_remaining INTEGER DEFAULT 5,
  logged_in_last_reset_at INTEGER,
  
  -- 付费配额
  paid_total INTEGER DEFAULT 0,
  paid_used INTEGER DEFAULT 0,
  paid_remaining INTEGER DEFAULT 0,
  paid_valid_until INTEGER,
  
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 7.3 套餐表 (pricing_plans)

```sql
CREATE TABLE pricing_plans (

  features TEXT,

  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);
```

### 7.4 订单表 (orders)
```sql
CREATE TABLE orders (
```
