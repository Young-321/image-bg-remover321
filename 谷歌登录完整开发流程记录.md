## 🔧 遇到的问题与解决方案

### 问题1：OAuth回调参数位置不匹配
**症状**: URL中有OAuth参数，但页面没反应  
**原因**: 参数在`?`后面（search），代码只检查`#`后面（hash）  
**解决**: 修改代码，先检查hash，没有再检查search
```typescript
// 先检查hash
const hash = window.location.hash.substring(1);
if (hash) {
  params = new URLSearchParams(hash);
} 
// 再检查search
else {
  const search = window.location.search.substring(1);
  if (!search) return;
  params = new URLSearchParams(search);
}
```

---

### 问题2：OAuth响应缺少id_token
**症状**: URL中只有access_token，没有id_token  
**原因**: OAuth scope配置问题，或者响应类型问题  
**解决**: 用access_token调用Google用户信息API
```typescript
fetch('https://www.googleapis.com/oauth2/v2/userinfo', {
  headers: {
    'Authorization': `Bearer ${accessToken}`,
  },
})
.then(response => response.json())
.then(userInfo => {
  // 使用获取到的用户信息
})
```

---

### 问题3：useEffect中不能用async/await
**症状**: 构建报错："await isn't allowed in non-async function"  
**原因**: useEffect的回调函数不能是async函数  
**解决**: 用Promise链替代async/await
```typescript
// ❌ 错误写法
useEffect(async () => {
  const response = await fetch(...);
}, []);

// ✅ 正确写法
useEffect(() => {
  fetch(...)
    .then(response => response.json())
    .then(data => { /* 处理数据 */ })
    .catch(err => { /* 处理错误 */ });
}, []);
```

---

### 问题4：刷新页面登录状态丢失
**症状**: 登录成功后刷新，登录信息消失  
**原因**: React state只在内存中，刷新后重置  
**解决**: 用localStorage持久化登录状态
```typescript
// 保存登录状态
const loginState = {
  isLoggedIn: true,
  userEmail: userInfo.email,
  userName: userInfo.name,
  timestamp: Date.now()
};
localStorage.setItem('bgRemover_loginState', JSON.stringify(loginState));

// 恢复登录状态
const savedLoginState = localStorage.getItem('bgRemover_loginState');
if (savedLoginState) {
  const state = JSON.parse(savedLoginState);
  setIsLoggedIn(true);
  setUserEmail(state.userEmail);
  setUserName(state.userName);
}

// 清除登录状态
localStorage.removeItem('bgRemover_loginState');
```

---

### 问题5：欢迎提示重复显示
**症状**: 有时会出现两个重叠的欢迎提示  
**原因**: state更新可能触发多次  
**解决**: 添加检查，防止重复显示
```typescript
if (!showWelcome) {
  setShowWelcome(true);
  setTimeout(() => setShowWelcome(false), 4000);
}
```

## 🔐 安全考虑

### localStorage安全
- ✅ 只存储非敏感信息（邮箱、姓名）
- ✅ 不存储access_token或密码
- ✅ 添加时间戳便于过期处理
- ⚠️ 注意：localStorage容易被XSS攻击，生产环境可考虑更安全的方案

### 成功的关键
