# AI 图片背景移除网站 - 需求文档

## 📋 项目概述

### 项目名称
AI 背景移除工具网站 (BG Remover)

### 项目目标
为普通用户提供简单、快速、免费的在线图片背景移除服务。

### 核心价值
- **简单易用**：上传图片 → 一键移除背景 → 下载结果
- **零成本**：个人使用基本免费
- **隐私保护**：图片不存储，内存处理
- **全球加速**：使用边缘计算，就近处理

## 🎯 用户画像

### 目标用户
- 普通用户（非专业设计师）
- 电商卖家（商品图片处理）
- 社交媒体用户（个人照片编辑）
- 学生/办公人员（文档图片处理）

### 用户需求
1. **简单操作**：不需要学习复杂软件
2. **快速处理**：等待时间短
3. **高质量结果**：边缘识别准确
4. **免费使用**：个人需求免费满足
5. **隐私安全**：图片不上传服务器保存

## 🏗️ 技术架构

### 总体架构
```
用户浏览器
    ↓
Cloudflare Pages (静态前端)
    ↓
Cloudflare Workers (边缘后端)
    ↓
Remove.bg API (AI处理)
```

### 技术栈
| 组件 | 技术选择 | 原因 |
|------|----------|------|
| **前端** | Next.js 14 + TypeScript + Tailwind CSS | 快速开发、类型安全、响应式设计 |
| **后端** | Cloudflare Workers | 边缘计算、无服务器、全球加速 |
| **AI服务** | Remove.bg API | 专业抠图、免费额度、稳定可靠 |
| **部署** | Cloudflare 生态 | 一体化管理、成本最低 |

### 架构优势
1. **零服务器成本**：使用 Cloudflare 免费额度
2. **全球加速**：300+ CDN 节点就近处理
3. **无状态设计**：图片内存处理，不存储
4. **弹性扩展**：自动应对流量高峰
5. **维护简单**：无需服务器运维

## 🔧 技术实现细节

### 前端实现 (Next.js)
```typescript
// 核心流程
1. 用户上传图片 → FileReader 读取为 DataURL
2. 显示预览 → <img src={dataURL}>
3. 点击处理 → FormData 包装图片
4. 调用 Worker → fetch(workerUrl, { body: formData })
5. 显示结果 → URL.createObjectURL(blob)
6. 下载保存 → <a download> 触发下载
```

### 后端实现 (Cloudflare Worker)
```typescript
// 处理流程
1. 接收 FormData → request.formData()
2. 验证文件 → 类型、大小检查
3. 调用 Remove.bg → fetch('https://api.remove.bg/v1.0/removebg')
4. 错误处理 → 友好的用户提示
5. 返回图片 → Response(arrayBuffer, image/png)
```

### API 集成 (Remove.bg)
- **免费额度**：50张/月
- **收费标准**：超出后 $0.29/张
- **响应格式**：PNG 图片
- **处理质量**：专业级抠图效果

## 🚀 部署方案

### 第一阶段：本地开发
```bash
# 前端开发服务器
pnpm dev          # http://localhost:3000

# Worker 开发服务器
pnpm worker:dev   # http://localhost:8787
```

### 第二阶段：部署到 Cloudflare

#### 1. Worker 部署
```bash
# 登录 Cloudflare
pnpm wrangler login

# 设置 API 密钥
pnpm wrangler secret put REMOVE_BG_API_KEY

# 部署 Worker
pnpm wrangler deploy
```

#### 2. 前端部署 (Cloudflare Pages)
1. 代码推送到 GitHub
2. Cloudflare Dashboard 创建 Pages 项目
3. 配置：
   - **构建命令**：`pnpm build`
   - **输出目录**：`out`
   - **环境变量**：`WORKER_URL` (可选)

#### 3. 域名配置 (可选)
1. DNS 添加 CNAME 记录
2. Pages 绑定自定义域名
3. Worker 设置路由规则

## 🔒 安全与隐私

### 隐私保护措施
1. **无持久化存储**：图片内存处理，处理完即释放
2. **无用户追踪**：不记录用户个人信息
3. **无数据备份**：不保存处理后的图片
4. **端到端加密**：所有传输使用 HTTPS

### 安全措施
1. **文件验证**：类型、大小、内容安全检查
2. **速率限制**：防止 API 滥用
3. **错误隔离**：单次失败不影响系统
4. **监控告警**：异常流量检测
